# Claude 独立分析摘要

## 结论

- 核心不是换皮，而是让 Outline 从扁平问卷恢复为“先全局、后分支”的空间结构。
- 推荐在现有 ID 与 ledger 上增加节点绑定元数据，而不是引入独立图数据库或 XMind 文件。
- 建议使用全局总图、业务分图和横切原则图；普通问题绑定稳定节点，横切议题记录影响节点集合。
- 强调节点 ID 不得因排序或重排漂移，布局状态不能污染 Outline digest。

## Codex 保留意见

- Claude 提议把三级 Formal Confirmation 和横切原则做成节点级独立确认。该建议会把当前整份 Outline 的一致授权拆碎，并造成跨分支组合状态，暂不采纳。
- Claude把 L1/L2/L3 同时当作导图深度和成熟度，混淆了现有合同。成熟度只能是整个 Outline 的状态，节点深度应另设字段。
- 横切决策不应机械地为每个受影响节点复制 `context_note`；应保留一个权威决策事件及其影响引用，避免一项政策变成多份可能漂移的事实。
